Step-by-Step Guide

Set It Up Yourself — Plain English, No Coding

Follow these 6 steps in order. Claude does the hard work automatically. You just answer questions about your firm.

🔒 About Security — It's Built In Automatically

You do not need to set up security separately. During Step 4 (the interview), Claude asks about your team and what data your tools will handle — then automatically generates a security configuration that covers credential protection, encryption, access controls, and audit logging. Every tool built afterward inherits these security standards automatically. The interview IS the security setup.

1

⏱ 10 minutes

Create Your Three Free Accounts

You need three online accounts. You may already have some of these. Create each one, then come back here.

① Anthropic (the company that makes Claude) — go to console.anthropic.com → Sign Up → go to API Keys in the left menu → click Create Key → name it "Law Firm" → copy the key and save it somewhere safe (you'll paste it in Step 2).

② GitHub — go to github.com → Sign Up with your firm email. This is where your tools are stored and backed up. Think of it as a filing cabinet for your AI tools.

③ Vercel — go to vercel.com → click Continue with GitHub. This makes your tools accessible on the internet so your staff can use them from any computer.

💡 Simple analogy: Claude Code builds the tool. GitHub stores it safely. Vercel runs it so your team can use it from anywhere.
2

⏱ 10 minutes

Install Claude Code on Your Computer

This is the only part that happens on your actual computer. You'll open a text window and paste two short commands — that's it.

On a Mac: Press Command + Space on your keyboard → type Terminal → press Enter. A window opens with a blinking cursor.

On Windows: Press the Windows key → type Command Prompt → press Enter.

Once the window is open, click inside it, paste this exactly, and press Enter:

npm install -g @anthropic/claude-code

Wait about 60 seconds while it installs. When you see a cursor again, paste this and press Enter:

claude

It will ask for your API key. Paste the one you saved from Step 1. Press Enter. You'll see a welcome message — Claude Code is ready.

⚠️ Seeing an error on the first command? You need to install Node.js first. Go to nodejs.org → click the big green "LTS" download button → run the installer → click Next through every screen → then try the command above again.
3

⏱ 2 minutes

Create Your Firm's Project Folder

This is the folder where all your tools will live — intake forms, research agents, document drafters, everything. Paste the right command for your computer and press Enter:

# On Mac — paste this: mkdir ~/LawFirmTools && cd ~/LawFirmTools && claude
# On Windows — paste this: mkdir C:\LawFirmTools && cd C:\LawFirmTools && claude

Claude Code is now open and pointed at your firm's folder. You're ready for the interview in Step 4.

Before writing any code, read security.config.md and confirm you will enforce

🔑
Credential Management
Before connecting to any external service. Verify this is working correctly.
Prevents API keys and passwords from being embedded in code. One leaked key can compromise every client, every system, and every integration your firm has. This skill establishes .env files, startup validation, and key rotation schedules. 
Read CLAUDE.md and security.config.md.

Audit this project for credential security:
1. Scan all source files — confirm no API keys, passwords, or tokens
   appear anywhere in the code or comments
2. Verify .env is in .gitignore and was never committed
3. Confirm all credentials load from process.env with startup validation
   (app should fail loudly if a required variable is missing)
4. Check .env.example exists with all required variable names but no values
5. Generate a key rotation schedule for all credentials in this project:
   what to rotate, how often, and the steps to do it safely

Report: pass/fail for each item. Fix any failures before continuing.
🛡️
Authentication + Access Control
Before any tool is used by more than one person.
Ensures only the right people access the right data. Connects to your firm's existing Google or Microsoft login so staff use accounts you already control. Role-based access enforced at the database level — not just the interface. When someone leaves the firm, disable their account once and they're locked out of everything. 
Read CLAUDE.md and security.config.md.

Implement authentication and role-based access control for this tool.

My firm uses: [Google Workspace / Microsoft 365] — specify one
Firm email domain: [@yourfirm.com]

Roles and permissions from security.config.md:

Requirements:

Encryption Layer
Before storing any client data in a database or file.
Protects client data if a server is compromised or a laptop is stolen. Encrypts sensitive fields at rest using AES-256 and enforces TLS 1.3 for all data in transit. Per-matter key derivation means each client's data is encrypted with a unique key. 
Read CLAUDE.md and security.config.md.

Implement the encryption layer for this tool.

Encrypt at rest:
- All fields listed as sensitive in security.config.md
- Use AES-256-GCM
- Encryption key from process.env.DATABASE_ENCRYPTION_KEY
- Derive a per-record key from master key + record ID
  so each record uses a distinct encryption key
- Store only ciphertext in database — never plaintext sensitive fields

Encrypt in transit:
- Enforce HTTPS on all endpoints — redirect HTTP to HTTPS
- Set minimum TLS 1.2, prefer 1.3
- Add security headers to all responses:
  Strict-Transport-Security, X-Content-Type-Options, X-Frame-Options

Generate the DATABASE_ENCRYPTION_KEY using:
openssl rand -base64 32
And tell me where to store it.

Verify: query the database directly and confirm sensitive fields
show ciphertext, not plaintext.
📋
Audit Logging
For any tool that reads, writes, or exports client data.
Your professional responsibility paper trail. Records who accessed what client data, when, and what they did — in an append-only log that cannot be altered. Required for state bar compliance (typically 5-7 year retention). Also your first line of defense if you ever face a breach allegation. 
Read CLAUDE.md and security.config.md.

Implement audit logging for this tool.

Every log record must capture:
- timestamp (with timezone)
- user_id and user_role at time of action
- user IP address
- session_id (groups related actions)
- action type (login / record_viewed / record_edited /
  document_downloaded / data_exported / admin_action)
- resource_type and resource_id (what was touched)
- matter_id if applicable
- outcome: success or failure
- failure_reason if failed

Storage requirements:
- Append-only: REVOKE UPDATE, DELETE on audit table from app user
- Retain for 7 years minimum
- Separate table/collection from application data

Alerts — email [alert email from security.config.md] when:
- 5+ failed logins from same IP in 10 minutes
- Any export affecting more than 20 records
- Any admin action (user creation, role change)
- Any access outside 7am-9pm local time to a matter flagged sensitive

Build an admin-only report endpoint that returns audit logs
as a downloadable CSV filtered by user, date range, or matter ID.
🛂
Input Validation
Before any form or file upload goes live — especially client-facing ones.
Every field a person can type into is a potential attack vector. Without validation, a single malicious submission can corrupt your database or expose other clients' data. This skill whitelists what's allowed and rejects everything else — including dangerous file uploads. 
Read CLAUDE.md and security.config.md.

Implement input validation for all forms and API endpoints in this tool.

Requirements:
- Validate ALL inputs server-side using Zod (TypeScript) or Pydantic (Python)
- Whitelist approach: define what IS allowed, reject everything else
- Text fields: trim whitespace, enforce max length, strip HTML tags
- Email: validate format, normalize to lowercase
- Phone: validate format, strip non-numeric except + and -
- Numbers: validate min/max range, reject NaN and Infinity
- Dates: validate as real calendar dates

File uploads:
- Accept PDF and DOCX only
- Validate by actual file content (magic bytes) — not just the extension
  (a renamed .exe should be rejected even if called document.pdf)
- Enforce 25MB maximum file size
- Strip dangerous characters from filenames

Error responses:
- Return generic user-friendly messages ("Invalid submission")
- Never expose stack traces, database errors, or internal details
- Log all validation failures with timestamp and IP address

Write tests:
- SQL injection attempt in every text field returns 400
- XSS attempt is sanitized or rejected
- .exe renamed to .pdf is rejected by content check
- File over 25MB is rejected
🔌
API Scope Control
Before connecting to Google Drive, your CRM, email, or any external service.
Every third-party integration is a potential breach point. This skill ensures every connection uses the minimum necessary permissions — so if a key is ever compromised, the damage is contained. A document reader only gets read access. A notification sender only gets send access. 
Read CLAUDE.md and security.config.md.

Configure API integrations with minimum necessary permissions.

For each integration in this project:
1. Request only the OAuth scope or API permission this tool actually needs
2. Read-only where writes are not required
3. Scope to specific resources — not entire accounts

Google Drive:
- Scope: drive.file (access only to files this app creates)
  OR drive.readonly (if only reading existing files)
- Restrict to folder ID in env GOOGLE_DRIVE_FOLDER_ID
- Never request full drive access

CRM (Clio / Lawmatics / MyCase):
- Intake tool: create_contacts, create_matters only (not billing, not delete)
- Status report: read_matters only
- Document sync: read_documents, create_documents (not delete)

Anthropic API:
- Never send full documents — extract only the specific text needed
- Strip client names and identifying information from prompts where possible
- Log all API calls with matter ID and token count

After building:
Run a permissions audit and output a table showing:
service | current scope | minimum possible scope | risk if key is stolen
Flag any integration where current scope exceeds minimum necessary.

The tests that prove the security layer actually works. Claude Code writes AND runs these tests itself — you just have to ask. Nothing goes live without passing this checklist. Catches the gaps that manual review misses.

Create Your Three Accounts

⏱ 10 minutes · One time only

You need three free accounts before anything else. Do them in this order — each one takes about 3 minutes.

Anthropic — the company that makes Claude

Go to console.anthropic.com → click Sign Up → enter your email and create a password → verify your email.

⚠️ This key is like a password. Don't share it, don't put it in an email, don't post it anywhere. You'll paste it into your terminal in Phase 2.

GitHub — where your tools are stored and backed up

Go to github.com → click Sign Up → use your firm email address → choose a username (e.g. smithlawfirm or your name) → choose the Free plan → verify your email.

Think of GitHub as a filing cabinet for your AI tools. Every version of every tool is saved here automatically. If your computer dies, everything is safe.

Vercel — where your tools go live on the internet

Install Claude Code on Your Computer

1

On a Mac: Press Command + Space on your keyboard at the same time → a search bar appears → type Terminal → press Enter. A window opens with a blinking cursor.

On Windows: Press the Windows key (bottom-left of keyboard) → type Command Prompt → press Enter. A black window opens.

Don't be intimidated by this window. You're just going to paste text into it and press Enter. That's it.

2

Install Node.js (if you haven't already)

Claude Code needs Node.js to run. To check if you have it: click inside the terminal window, paste this, and press Enter:

node --version

If you see a number like v20.0.0 — you're good, skip to Step 3.

If you see an error — go to nodejs.org → click the big green LTS button → download and run the installer → click Next through every screen → when done, close it and reopen your terminal.

3

Install Claude Code

Click inside the terminal, paste this exactly, and press Enter:

npm install -g @anthropic/claude-code

Wait about 60 seconds. You'll see text scrolling. When you see the cursor appear again (a $ or > symbol), it's done.

4

Connect Claude Code to your Anthropic account

Paste this and press Enter:

claude

It will ask for your API key. Paste the key you saved in Phase 1. Press Enter. You'll see a welcome message — Claude Code is installed and ready.

# On Mac: mkdir ~/LawFirmTools && cd ~/LawFirmTools && claude
